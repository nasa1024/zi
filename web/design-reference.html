THREE PILLARS

三大支柱，
缺一不可。

记忆让它「记得住」，一致性让它「不出错」，工艺让它「更好看」。三者正交，各司其职。

01

记忆

分层记忆 + 实体优先召回。软记忆走 RAG，硬状态进关系表。要用什么设定，它先帮你捞回来。

一致性

THE PIPELINE

一条流水线，五道工序

从规划到入库，每一章都跑同一条确定性管线。唯一的「晋升闸门」决定双模式如何分叉。

STAGE 01

plan

规划

定本章目标、beats、价值轴与钩子。

→
STAGE 02

recall

召回

实体优先捞回相关 canon 与历史片段。

→
STAGE 03

draft

起草

as-of 注入约束，LLM 写出正文初稿。

→
STAGE 04

check

校验

确定性 validator + 工艺检查双重过滤。

→
STAGE 05

gate

闸门

晋升 canon 或入审核队列——双模式在此分叉。

⚙️ 崩溃也不怕：L0 原子写入（temp→fsync→rename）+ pipeline_run 状态机，下次启动自动扫描残留、回滚到一致态。
